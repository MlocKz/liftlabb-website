**2. Create `components/Features.tsx`**

A "use client" component using Motion variants for staggered scroll-triggered entrance animation. Uses the container/item variant pattern -- parent defines stagger timing via `delayChildren: stagger(0.1)`, children inherit via variant propagation.

```tsx
"use client"
import { motion, stagger } from "motion/react"
import { features } from "@/lib/content"
import FeatureIcon from "@/components/FeatureIcon"

const containerVariants = {
  hidden: {},
  visible: {
    transition: {
      delayChildren: stagger(0.1),
    },
  },
}

const cardVariants = {
  hidden: { opacity: 0, y: 30 },
  visible: {
    opacity: 1,
    y: 0,
    transition: { duration: 0.5, ease: "easeOut" },
  },
}

export default function Features() {
  return (
    <section id="features" className="py-24 px-6">
      <div className="max-w-6xl mx-auto">
        <motion.h2
          initial={{ opacity: 0, y: 20 }}
          whileInView={{ opacity: 1, y: 0 }}
          viewport={{ once: true }}
          transition={{ duration: 0.5, ease: "easeOut" }}
          className="text-3xl md:text-4xl font-bold text-center mb-16"
        >
          Everything you need to train smarter
        </motion.h2>
        <motion.div
          variants={containerVariants}
          initial="hidden"
          whileInView="visible"
          viewport={{ once: true, margin: "-100px" }}
          className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6"
        >
          {features.map((feature) => (
            <motion.div
              key={feature.title}
              variants={cardVariants}
              className="bg-card border border-border rounded-card p-6 transition-all duration-300 hover:shadow-[0_0_20px_rgba(74,222,128,0.3)] hover:-translate-y-1 hover:border-accent/50"
            >
              <FeatureIcon name={feature.icon} />
              <h3 className="text-lg font-bold text-text mb-2">{feature.title}</h3>
              <p className="text-muted text-sm leading-relaxed">{feature.description}</p>
            </motion.div>
          ))}
        </motion.div>
      </div>
    </section>
  )
}
```

Key details:
- `stagger(0.1)` imported from "motion/react" (modern API, not deprecated staggerChildren)
- `viewport={{ once: true, margin: "-100px" }}` triggers animation when section is 100px into viewport, plays once only (Pitfall 5 prevention)
- Hover effects: green glow shadow `rgba(74,222,128,0.3)` matching accent #4ade80, lift via `-translate-y-1`, border highlight via `border-accent/50`
- `transition-all duration-300` covers both transform and shadow transitions on hover
- Cards do NOT have fixed heights -- content flows naturally (Pitfall 4 prevention)
- Grid: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` produces 1-col mobile, 2-col tablet (768px+), 3-col desktop (1024px+)

**1. Create `components/PhoneMockup.tsx`**

A reusable CSS-only phone frame. No "use client" needed (pure presentational JSX). This component will also be reusable by the hero section (Phase 3) if a shared phone frame is desired.

```tsx
// components/PhoneMockup.tsx
export default function PhoneMockup({
  children,
  className = "",
}: {
  children: React.ReactNode
  className?: string
}) {
  return (
    <div
      className={`relative rounded-[36px] border-[3px] border-border bg-card overflow-hidden ${className}`}
      style={{ aspectRatio: "9 / 19.5" }}
    >
      {/* Notch */}
      <div className="absolute top-0 left-1/2 -translate-x-1/2 w-[100px] h-[28px] bg-bg rounded-b-2xl z-10" />
      {/* Screen content */}
      <div className="w-full h-full">
        {children}
      </div>
    </div>
  )
}
```

Key details:
- Width is NOT hardcoded -- pass it via className prop (e.g., `w-[260px]`). This makes it reusable at different sizes.
- `overflow-hidden` on the outer container prevents content from bleeding past rounded corners (Pitfall 3 prevention)
- Aspect ratio 9/19.5 matches modern iPhone proportions
- Border radius 36px matches modern phone curves
- Notch: centered pill at top using `bg-bg` (page background color) so it blends seamlessly

**2. Create `components/Screenshots.tsx`**

A "use client" component with a horizontal scroll-snap gallery. Uses Motion for the section heading entrance only. The gallery itself is pure CSS -- no JS scroll library needed.

```tsx
"use client"
import { motion } from "motion/react"
import { screenshots } from "@/lib/content"
import PhoneMockup from "@/components/PhoneMockup"

export default function Screenshots() {
  return (
    <section id="screenshots" className="py-24 px-6 overflow-hidden">
      <div className="max-w-6xl mx-auto">
        <motion.h2
          initial={{ opacity: 0, y: 20 }}
          whileInView={{ opacity: 1, y: 0 }}
          viewport={{ once: true }}
          transition={{ duration: 0.5, ease: "easeOut" }}
          className="text-3xl md:text-4xl font-bold text-center mb-16"
        >
          See it in action
        </motion.h2>
        <div
          className="flex gap-6 overflow-x-auto snap-x snap-mandatory scroll-smooth pb-4 [scrollbar-width:none] [&::-webkit-scrollbar]:hidden"
        >
          {/* Left spacer for centering on desktop */}
          <div className="shrink-0 w-[max(0px,calc((100%-5*260px-4*24px)/2))]" aria-hidden="true" />
          {screenshots.map((s) => (
            <div key={s.name} className="snap-center shrink-0">
              <PhoneMockup className="w-[260px]">
                <div
                  className="w-full h-full flex items-center justify-center"
                  style={{ backgroundColor: s.color }}
                >
                  <span className="text-text/70 text-sm font-medium">{s.name}</span>
                </div>
              </PhoneMockup>
            </div>
          ))}
          {/* Right spacer for centering on desktop */}
          <div className="shrink-0 w-[max(0px,calc((100%-5*260px-4*24px)/2))]" aria-hidden="true" />
        </div>
      </div>
    </section>
  )
}
```

Key details:
- `snap-x snap-mandatory` for reliable snapping on iOS (not proximity -- Pitfall 2 prevention)
- `scroll-smooth` for smooth scroll behavior
- `[scrollbar-width:none] [&::-webkit-scrollbar]:hidden` hides scrollbar on all browsers (cleaner look)
- `shrink-0` on each item prevents flex from collapsing the phone mockups
- `snap-center` centers each phone in the viewport when snapping
- Spacer divs with `calc()` center the gallery on desktop when all items fit. On mobile where items overflow, the spacers collapse to 0.
- `overflow-hidden` on the section prevents horizontal scroll bleed to the page body
- Section has `id="screenshots"` for potential nav link targeting
- Phone mockups are 260px wide with 24px (gap-6) between them
