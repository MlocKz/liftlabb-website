**2. Create components/Navbar.tsx**

Build a client component with "use client" directive containing:

**Imports:**
- `useState`, `useEffect` from "react"
- `AnimatePresence`, `motion` from "motion/react"
- `Image` from "next/image"
- `siteConfig`, `navLinks` from "@/lib/content"

**State:**
- `scrolled` (boolean, default false) -- toggles glassmorphic background
- `mobileOpen` (boolean, default false) -- toggles mobile menu overlay

**Scroll detection (useEffect):**
- Add scroll event listener with `{ passive: true }` (CRITICAL for performance)
- Threshold: `window.scrollY > 50`
- Clean up listener in return function to prevent memory leak

**JSX structure -- the `<nav>` wrapper:**
- `fixed top-0 left-0 right-0 z-50` for sticky positioning
- `transition-all duration-300` for smooth glassmorphism fade
- When `scrolled` is true: `bg-bg/80 backdrop-blur-md border-b border-border/50 shadow-lg shadow-black/10`
- When `scrolled` is false: `bg-transparent`
- Inner container: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- Flex row: `flex items-center justify-between h-16`

**JSX structure -- logo (left side):**
- Wrap in `<a href="#">` (scrolls to top)
- `flex items-center gap-2.5 group` for layout and hover group
- `<Image>` component: `src="/LiftLabb-Logo.png"` `alt="LiftLabb"` `width={32}` `height={32}` `className="rounded-lg"` `priority` (above-the-fold, prevents layout shift)
- Text "LiftLabb" in a `<span>`: `text-accent font-bold text-lg tracking-tight`
- Use `siteConfig.name` for the text, not a hardcoded string

**JSX structure -- desktop nav links (center/right, hidden on mobile):**
- Wrapper: `hidden md:flex items-center gap-1`
- Map over `navLinks` to render `<a>` tags (NOT `<Link>` from next/link -- anchor links must use plain `<a>` to work with scroll-behavior: smooth)
- Each link: `text-sm text-muted hover:text-text px-3 py-2 rounded-lg hover:bg-white/5 transition-colors duration-200`
- The hover:bg-white/5 gives a subtle highlight on hover that feels interactive without being heavy

**JSX structure -- Launch App CTA button (desktop):**
- Plain `<a>` tag (not next/link)
- `href={siteConfig.appUrl}` `target="_blank"` `rel="noopener noreferrer"`
- Styling: `ml-2 bg-accent text-bg text-sm font-bold px-5 py-2 rounded-full hover:bg-accent/90 hover:shadow-lg hover:shadow-accent/20 active:scale-[0.98] transition-all duration-200`
- Use `rounded-full` (pill shape) to differentiate the CTA from nav links -- it should pop visually
- Text: "Launch App" (from research requirements, hardcoded is fine here)

**JSX structure -- hamburger button (mobile only):**
- `<button>` with `onClick={() => setMobileOpen(!mobileOpen)}`
- `md:hidden` to hide on desktop
- `relative w-10 h-10 flex items-center justify-center rounded-lg hover:bg-white/5 transition-colors`
- `aria-label={mobileOpen ? "Close menu" : "Open menu"}`
- `aria-expanded={mobileOpen}`
- Render an animated hamburger icon using three `<span>` bars instead of SVG for smoother animation:
  - Three bars: `block h-0.5 w-5 bg-text transition-all duration-300 absolute`
  - Bar 1 (top): When closed: `translate-y-[-6px]`. When open: `translate-y-0 rotate-45`
  - Bar 2 (middle): When closed: `opacity-100`. When open: `opacity-0`
  - Bar 3 (bottom): When closed: `translate-y-[6px]`. When open: `translate-y-0 -rotate-45`
  - This creates the classic hamburger-to-X cross-fade animation purely with CSS transitions

**JSX structure -- mobile menu overlay (below nav):**
- Wrap in `<AnimatePresence>` (required for exit animations)
- Conditionally render `{mobileOpen && (<motion.div ... />)}`
- `motion.div` props:
  - `initial={{ opacity: 0, y: -8 }}`
  - `animate={{ opacity: 1, y: 0 }}`
  - `exit={{ opacity: 0, y: -8 }}`
  - `transition={{ duration: 0.2, ease: "easeOut" }}`
- Outer classes: `md:hidden absolute top-full left-0 right-0 bg-bg/95 backdrop-blur-lg border-b border-border/50`
- Inner padding: `px-4 py-3 flex flex-col`
- Map over `navLinks` rendering `<a>` tags:
  - Each: `text-muted hover:text-text text-base py-3 px-3 rounded-lg hover:bg-white/5 transition-colors border-b border-border/30 last:border-b-0`
  - CRITICAL: `onClick={() => setMobileOpen(false)}` on each link to close menu after navigation
- Launch App CTA at bottom of mobile menu:
  - `<a>` tag with same href/target/rel as desktop version
  - `mt-2 bg-accent text-bg font-bold py-3 rounded-full text-center hover:bg-accent/90 transition-colors`
  - Text: "Launch App"

**Design decisions and rationale:**
- `rounded-full` on CTA buttons (pill shape) creates stronger visual hierarchy than `rounded-card`
- `hover:bg-white/5` on nav links provides subtle interactivity without competing with the CTA
- `shadow-lg shadow-black/10` on scrolled state adds depth without being heavy on dark backgrounds
- `hover:shadow-accent/20` on CTA gives a green glow hover effect that matches the brand
- `active:scale-[0.98]` on CTA provides tactile click feedback
- Animated hamburger bars (not SVG swap) feels more polished -- the morphing transition is smooth
- `border-b border-border/30` separators in mobile menu give structure without heaviness
- `h-16` (64px) navbar height is standard and matches the `pt-16` offset on the page

**What NOT to do:**
- Do NOT import `Link` from `next/link` (anchor links must use `<a>` for smooth scroll to work -- see PITFALLS.md #8)
- Do NOT use Motion `useScroll` / `useTransform` for scroll detection (overkill for a boolean toggle -- see RESEARCH.md Pattern 1)
- Do NOT forget `{ passive: true }` on the scroll listener (causes scroll jank without it)
- Do NOT forget the cleanup return in useEffect (memory leak -- see RESEARCH.md Pitfall 4)
- Do NOT apply `backdrop-blur` to the mobile menu at full blur -- use `backdrop-blur-lg` sparingly (PERF-02)

**1. Add scroll-margin-top to app/globals.css**

Add this rule AFTER the existing `html { scroll-behavior: smooth; }` block:

```css
/* Offset anchor scroll position to clear fixed navbar (h-16 = 64px, + 16px breathing room) */
section[id] {
  scroll-margin-top: 5rem;
}
```
