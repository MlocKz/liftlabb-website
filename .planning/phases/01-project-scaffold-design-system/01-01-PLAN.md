Run the create-next-app command to scaffold the project in the current directory. The directory already contains `.planning/`, `.git/`, `docs/`, and `LiftLabb Logo.png` -- create-next-app will add its files alongside these.

```bash
npx create-next-app@15 . --typescript --tailwind --eslint --app --src-dir=false --import-alias="@/*"
```

IMPORTANT flags:
- `@15` pins to Next.js 15 (NOT 16 -- Next.js 16 has breaking changes we do not need)
- `--src-dir=false` keeps files in the root (no src/ directory)
- `--tailwind` scaffolds Tailwind v4 CSS-first config (since Next.js 15.2)

After scaffolding, verify and clean up:
1. Check `app/globals.css` uses `@import "tailwindcss"` (v4 syntax), NOT `@tailwind base; @tailwind components; @tailwind utilities;` (v3 syntax). If v3 syntax, replace with `@import "tailwindcss";`.
2. Check `postcss.config.mjs` uses `"@tailwindcss/postcss": {}` plugin. If it references the old `tailwindcss` plugin, update to `@tailwindcss/postcss`.
3. DELETE `tailwind.config.ts` or `tailwind.config.js` if one was generated -- Tailwind v4 uses CSS-first config only. A JS config file would be silently ignored.
4. Ensure `next.config.ts` does NOT have `output: 'export'` -- this would disable next/image optimization. The default mode is correct for Vercel deployment.

Then install the Motion animation library:
```bash
npm install motion
```

Do NOT install `framer-motion` (old package name, rebranded to `motion` in late 2024).

Verify `package.json` contains:
- `next` at 15.x (not 16.x)
- `tailwindcss` at 4.x
- `motion` at 12.x
