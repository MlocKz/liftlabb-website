**Step 1: Configure globals.css with LiftLabb design tokens.**

Replace the contents of `app/globals.css` with the Tailwind v4 CSS-first design system. This uses `@theme` for static color/radius values and `@theme inline` for the font variable (which is a runtime CSS variable injected by next/font/local).

```css
/* app/globals.css */
@import "tailwindcss";

/* Static design tokens -- use @theme */
@theme {
  --color-bg: #0a0a0a;
  --color-card: #1a1a1a;
  --color-card-2: #222222;
  --color-text: #e5e5e5;
  --color-muted: #999999;
  --color-accent: #4ade80;
  --color-yellow: #facc15;
  --color-red: #f87171;
  --color-blue: #60a5fa;
  --color-border: #333333;
  --radius-card: 12px;
}

/* Dynamic font variable from next/font -- MUST use @theme inline */
/* CRITICAL: Use @theme inline (not @theme) because var(--font-circular) is */
/* injected at runtime by next/font/local. @theme alone cannot resolve runtime vars. */
@theme inline {
  --font-sans: var(--font-circular), -apple-system, BlinkMacSystemFont, 'SF Pro', sans-serif;
}

/* Base styles */
body {
  background-color: var(--color-bg);
  color: var(--color-text);
}

/* Smooth scrolling for anchor links */
html {
  scroll-behavior: smooth;
}
```

This generates Tailwind utilities: `bg-bg`, `bg-card`, `bg-card-2`, `text-text`, `text-muted`, `text-accent`, `text-yellow`, `text-red`, `text-blue`, `border-border`, `rounded-card`, `font-sans`.

**Step 2: Configure layout.tsx with next/font/local.**

Replace `app/layout.tsx` with the root layout that loads Circular Std via `next/font/local`:

```tsx
import type { Metadata } from 'next'
import localFont from 'next/font/local'
import './globals.css'

const circularStd = localFont({
  src: './fonts/CircularStd-Medium.ttf',
  weight: '500',
  style: 'normal',
  display: 'swap',
  variable: '--font-circular',
})

export const metadata: Metadata = {
  title: 'LiftLabb - Track your gains. Ditch the spreadsheet.',
  description: 'The workout tracker built for lifters who want to get stronger.',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={circularStd.variable}>
      <body className="bg-bg text-text font-sans antialiased min-h-screen">
        {children}
      </body>
    </html>
  )
}
```

Key details:
- `variable: '--font-circular'` creates a CSS custom property on the `<html>` element
- `className={circularStd.variable}` on `<html>` injects the CSS variable
- The `@theme inline` in globals.css connects `--font-sans` to `var(--font-circular)`, making `font-sans` utility use Circular Std
- `display: 'swap'` ensures text is visible immediately while font loads (prevents layout shift)
- Body uses `bg-bg text-text font-sans antialiased` to apply the design system defaults

**Step 2: Create components/TestAnimation.tsx.**

Create a client component that demonstrates Motion's `whileInView` scroll-triggered animation using the LiftLabb design tokens:

```tsx
"use client"
import { motion } from "motion/react"

export default function TestAnimation() {
  return (
    <motion.div
      initial={{ opacity: 0, y: 30 }}
      whileInView={{ opacity: 1, y: 0 }}
      viewport={{ once: true }}
      transition={{ duration: 0.6, ease: "easeOut" }}
      className="p-8 bg-card rounded-card border border-border max-w-md"
    >
      <h2 className="text-xl font-bold text-accent mb-2">Motion Works!</h2>
      <p className="text-muted text-sm">
        This element animated into view using Motion whileInView.
      </p>
    </motion.div>
  )
}
```

Key details:
- `"use client"` directive is required because Motion uses browser APIs
- Import from `"motion/react"` (NOT `"framer-motion"`)
- `viewport={{ once: true }}` prevents animation from replaying on re-scroll
- Uses design token classes: `bg-card`, `rounded-card`, `border-border`, `text-accent`, `text-muted`
