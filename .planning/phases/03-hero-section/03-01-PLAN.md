  <action>
Create `components/Hero.tsx` as a `"use client"` component (~100-130 lines). This single file contains the entire hero section. Do NOT extract sub-components (PhoneMockup, CTAButton, etc.) -- premature abstraction for a one-use hero.

**Imports:**
```tsx
"use client"
import { motion, stagger } from "motion/react"
import { siteConfig } from "@/lib/content"
```

**Motion animation variants (defined outside the component):**

- `containerVariants`: `hidden: {}`, `visible: { transition: { delayChildren: stagger(0.15) } }` -- orchestrates staggered entrance of all children. Use `stagger()` function (NOT deprecated `staggerChildren`).
- `itemVariants`: `hidden: { opacity: 0, y: 20 }`, `visible: { opacity: 1, y: 0, transition: { duration: 0.6, ease: "easeOut" } }` -- each child fades up 20px.
- Only the root `motion.section` gets `initial="hidden" animate="visible"`. Children get ONLY `variants={itemVariants}` -- no `initial`/`animate` on children (would break stagger orchestration, see Research Pitfall 5).

**Section layout:**
```
<motion.section id="hero" initial="hidden" animate="visible" variants={containerVariants}
  className="relative min-h-[100dvh] flex items-center overflow-hidden">
```
- `min-h-[100dvh]` (NOT `min-h-screen`) -- accounts for mobile browser chrome (Research Pitfall 3).
- `overflow-hidden` -- prevents glow bleed causing horizontal scrollbar.
- `id="hero"` -- for potential future nav anchoring.

**Radial green glow (background layer, HERO-07):**
Inside the section, before the content grid. Use blurred absolute div approach (Research Option A):
```tsx
<div className="absolute inset-0 pointer-events-none" aria-hidden="true">
  <div className="absolute top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2
                  w-[600px] h-[400px] lg:w-[800px] lg:h-[600px] rounded-full
                  bg-accent/10 blur-[120px]" />
</div>
```
- `bg-accent/10` -- accent green (#4ade80) at 10% opacity for subtle ambient glow.
- `blur-[120px]` -- large blur radius for soft spread. Static (not animated) -- no GPU cost after initial paint.
- `pointer-events-none` + `aria-hidden="true"` -- non-interactive decorative element.
- Responsive sizing: 600x400 mobile, 800x600 desktop.

**Two-column content grid:**
```tsx
<div className="relative z-10 max-w-7xl mx-auto px-6 w-full
                flex flex-col lg:flex-row items-center gap-12 lg:gap-16
                pt-32 pb-16 lg:py-0">
```
- `flex-col lg:flex-row` -- stacks on mobile (text first), side-by-side on desktop (1024px+).
- `pt-32` -- top padding accounts for future navbar height on mobile.
- `lg:py-0` -- on desktop, flex items-center vertically centers within min-h-[100dvh].
- `z-10` -- above the glow layer.

**Text column (left on desktop, top on mobile):**
Wrap in `<div className="flex-1 text-center lg:text-left">`.

- **Headline (HERO-01):** `<motion.h1 variants={itemVariants}>` with `{siteConfig.tagline}`. Classes: `text-4xl sm:text-5xl lg:text-6xl font-bold text-text leading-tight tracking-tight`. This is the page's only `h1` -- critical for SEO heading hierarchy.
- **Subtext (HERO-02):** `<motion.p variants={itemVariants}>` with `{siteConfig.description}`. Classes: `mt-6 text-lg sm:text-xl text-muted max-w-lg mx-auto lg:mx-0`. Max-width keeps line length readable. `mx-auto lg:mx-0` centers on mobile, left-aligns on desktop.
- **CTA buttons (HERO-03, HERO-04):** `<motion.div variants={itemVariants}>` wrapping both buttons in a flex container: `mt-8 flex flex-col sm:flex-row gap-4 justify-center lg:justify-start`.

  **Primary CTA "Get Started Free" (HERO-03):**
  Use `<a>` tag (NOT `<Link>` -- external URL). `href={siteConfig.appUrl}`.
  Classes: `inline-flex items-center justify-center px-8 py-3.5 bg-accent text-bg font-bold rounded-full text-lg hover:brightness-110 transition-all`.
  - `bg-accent text-bg` -- green button with dark text, high contrast (~7.5:1 ratio).
  - `rounded-full` -- pill shape for modern CTA aesthetic.
  - `hover:brightness-110` -- subtle brightness lift on hover (GPU-friendly, no layout shift).
  - `px-8 py-3.5` -- generous padding for large click target.

  **Secondary CTA "See Features" (HERO-04):**
  Use `<a>` tag (NOT `<Link>` -- same-page anchor, Pitfall 8). `href="#features"`.
  Classes: `inline-flex items-center justify-center px-8 py-3.5 border border-border text-text rounded-full text-lg hover:border-accent hover:text-accent transition-colors`.
  - Ghost/outline style -- doesn't compete with primary CTA.
  - Smooth scroll works automatically via `scroll-behavior: smooth` in globals.css.

**Phone mockup column (right on desktop, bottom on mobile, HERO-05):**
Wrap in `<motion.div variants={itemVariants} className="flex-1 flex justify-center">`.

CSS-drawn phone frame (Flowbite/Themesberg pattern adapted for dark theme):
```tsx
<div className="relative border-[14px] border-card-2 bg-card-2
                rounded-[2.5rem] h-[500px] w-[245px]
                sm:h-[600px] sm:w-[300px] shadow-xl">
  {/* Notch */}
  <div className="absolute top-0 left-1/2 -translate-x-1/2

                  h-[46px] w-[3px] bg-card-2 rounded-l-lg" />

  {/* Power button (right side) */}
  <div className="absolute -right-[17px] top-[142px]

  {/* Screen content area */}

                  flex items-center justify-center">
    <span className="text-muted text-sm">App Preview</span>
  </div>
</div>
```
- `border-card-2` (#222222) -- dark bezel, subtle against #0a0a0a bg.
- Responsive sizing: 245x500 on mobile, 300x600 on sm+.
- Screen area uses `bg-bg` to match page background for seamless feel.
- Placeholder text "App Preview" will be swapped for real screenshot (next/image) in v2.

**Animation performance (PERF-03):** Only `opacity` and `y` (translate) are animated -- both GPU-accelerated. No layout property animations.

**What NOT to do:**
- Do NOT extract PhoneMockup, HeroCTA, or HeroHeadline sub-components.
- Do NOT use `<Link>` from next/link for either CTA.
- Do NOT use `staggerChildren` (deprecated) -- use `delayChildren: stagger(0.15)`.
- Do NOT add `initial`/`animate` to child motion elements (breaks variant propagation).
- Do NOT animate the glow (static only -- no ongoing GPU cost).
- Do NOT use `min-h-screen` (100vh fails on mobile -- use 100dvh).
